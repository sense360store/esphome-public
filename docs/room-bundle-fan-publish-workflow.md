The workflow is `workflow_dispatch` only and defaults to `dry_run: true`.
Dry-run validates the publish set and the generated release body, then exits
without building firmware or publishing anything. A deliberate dispatch with
`dry_run: false` builds the selected target(s), renames each output to its
expected `-v1.0.0-preview.bin` name, validates the exact output set, and attaches
the `.bin` files plus checksums and a build-info `manifest.json` to the shared
`v1.0.0-preview` preview release — the single preview release for every preview
artifact (room-bundle + LED + fan), per
[`RELEASE-PREVIEW-FAN-SHARED-TAG-001`](release-preview-fan-publish-results.md).

## Source of truth

The workflow matrix is generated by
[`scripts/validate_room_bundle_fan_publish.py`](../scripts/validate_room_bundle_fan_publish.py)
from:

- [`config/room-bundle-fan-variants.json`](../config/room-bundle-fan-variants.json) — the canonical per-variant source of truth (status, posture, FanDAC address requirement, stable blockers).
- [`config/compile-only-targets.json`](../config/compile-only-targets.json) — the compile-evidence ledger (each of the five carries `compile_validation_status: validated-full-compile` + a `compile_evidence` block citing run `26913592989`).

It does **not** read [`config/webflash-builds.json`](../config/webflash-builds.json)
as a matrix source and does **not** add fan rows to it — that file remains the
WebFlash release-eligibility ledger, and the fan-token guardrail stays intact.

## Workflow behavior

- `release_target=all-room-bundle-fan-previews` builds all five room-bundle fan targets.
- `release_target=<one room-bundle fan config>` scopes to a single target.
- `Ceiling-POE-VentIQ-FanTRIAC-RoomIQ` is **not selectable** and the helper fails
  closed if asked for it, because `HW-005` still blocks buildability and there is
  no compile proof.
- The top-level workflow token scope is `contents: read`; only the `publish` job
  requests `contents: write`, and only behind `dry_run: false`.
- The release vehicle is the shared `v1.0.0-preview` preview release. The publish
  step **attaches assets only** via `softprops/action-gh-release` (upsert); it
  does **not** create a new release / tag. WebFlash import eligibility is
  controlled separately by WebFlash import policy, so attaching an artifact here
  does not imply WebFlash import, Simple-install exposure, stable status,
  recommendation, default, or buyability.

## FanDAC IC2 `0x5A` address requirement (`FANDAC-I2C-ADDR-001` stays pending)

The two FanDAC configs — `Ceiling-POE-VentIQ-FanDAC-RoomIQ` and
`Ceiling-POE-AirIQ-FanDAC-RoomIQ` — compiled **with** the GP8403 IC2 address
override `fan_dac_2_i2c_address: "0x5A"`. The validator enforces, and the release
body records, the address policy:

| Address | Role |
|---|---|
| GP8403 IC1 | `0x58` (package default) |
| GP8403 IC2 | `0x5A` (relocated off the `0x59` default) |
| GP8403 `0x59` | **Forbidden** with VentIQ / AirIQ (collides with the SGP41 @ `0x59`) |
| SGP41 | `0x59` |

This is **compile-time configuration only**. The physical IC2 DIP-switch (SW2) →
I²C address mapping is **NOT bench verified**; the DIP-position → address truth
table stays owed by `FANDAC-I2C-ADDR-001`, which remains **pending**. **No FanDAC
hardware verification is claimed** by this workflow.

## Release-tag confirmation guard (`RELEASE-PREVIEW-FAN-PUBLISH-TAG-GUARD-001`)

The shared `v1.0.0-preview` preview release is the **frictionless default**, and
this workflow reuses the existing shared tag confirm-gate (it does not invent a
new one):

- A **"Guard release tag"** step in the `generate-matrix` job runs
  `scripts/validate_room_bundle_fan_publish.py --validate-release-tag` and
  **fails the whole run before any build** when the tag is not allowed.
- `release_tag` defaults to the shared `v1.0.0-preview`; that default needs no
  confirmation.
- Any **other** tag requires `confirm_tag_override=true`. This is a deliberate
  "are you sure?" gate against an accidental dispatch to the wrong tag (a typo,
  the stable release, or a stray new release).

## Guardrails

This PR does **not** publish firmware, run the workflow, create a GitHub Release
or tag, commit a `.bin`, write repo-root `manifest.json`, write
`firmware/sources.json`, touch the WebFlash repo, add any
`config/webflash-builds.json` row, include TRIAC, mark any fan stable /
recommended / default / buyable, mark FanDAC hardware-verified, change Simple
install, alter the single-driver manual-preview fan publish behavior, or claim
hardware / bench / compliance / safety / commercial proof.

The eventual run remains queued as `ROOM-BUNDLE-FAN-PUBLISH-RUN-001`.
