ESPHome Configuration

Customize your Sense360 firmware with YAML packages, thresholds, and advanced settings.

Configuration Approaches

Choose the approach that matches your needs:

Product Files

Pre-built configurations for standard setups. Everything included and tested.

Simple

Individual Packages

Pick and choose specific modules for custom hardware combinations.

Moderate

External Components

Only C++ drivers. Write all YAML yourself. Full control.

Advanced

Approach 1: Product Files (Recommended)

Use a single pre-built product file. Best for most users.

packages:
  sense360_firmware:
    url: https://github.com/sense360store/esphome-public
    ref: main
    files:
      - products/sense360-core-ceiling.yaml
    refresh: 1d

Approach 2: Individual Packages

Build custom configurations by selecting specific packages.

packages:
  sense360_base:
    url: https://github.com/sense360store/esphome-public
    ref: main
    files:
      # Base (required)
      - packages/base/wifi.yaml
      - packages/base/api_encrypted.yaml
      - packages/base/ota.yaml
      - packages/base/time.yaml

      # Hardware
      - packages/hardware/sense360_core_ceiling.yaml
      - packages/hardware/led_ring_ceiling.yaml

      # Expansions (choose modules)
      - packages/expansions/presence_ceiling.yaml

      # Features
      - packages/features/presence_basic_profile.yaml
    refresh: 1d

Approach 3: External Components Only

For experts who want just the C++ drivers and will write all YAML configuration themselves.

external_components:
  - source:
      type: git
      url: https://github.com/sense360store/esphome-public
      ref: main
    components: [ld2412, ld24xx]
    refresh: 1d

# You must define ALL configuration yourself:
# UART, sensors, LEDs, automations, etc.
Warning: This approach only provides C++ component code, not YAML configuration. Use only if you need complete control over your firmware.

Air Quality Thresholds

Customize when indicators show good, moderate, or unhealthy levels.

CO2 (Carbon Dioxide)

Substitution Default Meaning
co2_good_limit 750 ppm Below = excellent
co2_moderate_limit 950 ppm 750-950 = acceptable
co2_unhealthy_limit 1400 ppm 950-1400 = needs ventilation

PM2.5 (Particulate Matter)

Substitution Default Meaning (WHO Guidelines)
sen55_pm2_5_good_limit 10 µg/m³ Below = safe
sen55_pm2_5_moderate_limit 25 µg/m³ 10-25 = acceptable short-term
sen55_pm2_5_unhealthy_limit 55 µg/m³ 25-55 = sensitive groups affected

Example: Custom Thresholds

substitutions:
  device_name: sense360-bedroom
  friendly_name: "Bedroom Sense360"

  # Stricter thresholds for sleep quality
  co2_good_limit: "600"
  co2_moderate_limit: "800"
  sen55_pm2_5_good_limit: "8"

Custom Automations

High CO2 Alert

binary_sensor:
  - platform: template
    name: "${friendly_name} High CO2 Alert"
    device_class: safety
    lambda: |-
      return id(scd4x_co2).state > 1200;

Room Occupancy

binary_sensor:
  - platform: template
    name: "${friendly_name} Occupied"
    device_class: occupancy
    lambda: |-
      return id(ld2450_zone1_target_count).state > 0;

Air Quality Status

text_sensor:
  - platform: template
    name: "${friendly_name} Air Quality"
    lambda: |-
      float co2 = id(scd4x_co2).state;
      float pm25 = id(sen55_pm_2_5).state;

      if (co2 > 1400 || pm25 > 55) return {"Poor"};
      if (co2 > 950 || pm25 > 25) return {"Unhealthy"};
      if (co2 > 750 || pm25 > 10) return {"Moderate"};
      return {"Good"};
    update_interval: 60s

    gateway: 192.168.10.1

    dns1: 192.168.10.1

Fallback Access Point

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password
  ap:
    ssid: "Sense360-Setup"
    password: "SetupPassword123"

Best Practices

Use Version Tags: Always use specific version tags (e.g., ref: v3.0.0) instead of main for production devices to ensure stability.
Test Incrementally: Change one setting at a time, flash, and verify before making additional changes.
Back Up Configurations: Save copies of working configurations before making major changes.
Never Commit Secrets: Keep passwords and API keys in secrets.yaml, never in your main configuration files.
