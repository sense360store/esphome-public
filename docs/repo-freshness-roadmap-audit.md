## 0. Methodology, provenance, and limits

**File-age basis.** The cloned snapshot's entire git history spans only **2026-05-20 → 2026-05-24**
(verified: `git log --reverse` oldest commit `2026-05-20`, newest `2026-05-24`). There are therefore
**no files older than 5 months by VCS metadata**, and filesystem mtimes are reset by clone and carry
no signal. The "~5-month-old source-of-truth" heuristic from the audit brief cannot be computed from
version control in this snapshot. Freshness was instead assessed by:

1. **Content currency** vs the latest landed PRs (#577 PRODUCT-DAC-001 → #581 COMPILE-STATUS-FLAGS-001).
2. **Relative last-touch lineage** (`git log -1 -- <file>`): files whose last touch traces to the
   PR #532 import layer (`2026-05-20`) are the stable foundation; files touched by #577–#581 are the
   active edge.
3. **Generated-file sync status**, verified by running the regeneration tests (see §9 Validation).
4. **Explicit `source_of_truth` / `lifecycle` markers** declared inside the config JSON headers.

**Provenance / tooling limits.**
- The audit brief names the companion repo `sense360store/WebFlash`, but this session's GitHub access
  is scoped to `sense360store/esphome-public` and `sense360store/esphome` only. A direct read of
  `sense360store/WebFlash` returned **access denied**. The local second checkout `/home/user/esphome`
  is an unrelated single-device config repo, not WebFlash. **Cross-repo drift against the WebFlash repo
  could not be verified directly** and is recorded as `NEEDS-TOOLING` → `WEBFLASH-DRIFT-001`.
- No Dependabot / code-scanning / secret-scanning **alert** API was available in this session (the
  available secret-scanner only scans content passed to it, not the repo's alert feed). Dependency /
  alert findings below are from **manual inspection only**; this audit therefore makes **no
  "no vulnerabilities" claim** (`NEEDS-TOOLING`).
- No Google Drive evidence (schematics / BOMs / gerbers) was retrieved; no such evidence is fabricated.
  Where hardware evidence is missing it is recorded as a gap for operator input.
