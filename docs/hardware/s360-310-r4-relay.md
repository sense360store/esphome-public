# S360-310-R4 Relay Pin / Package Mapping Audit (HW-PINMAP-310)

## Status

**Status: partial — schematic evidence available; package reconciliation pending.**

This document is the **HW-PINMAP-310** audit record for the
`Sense360 Relay` board, revision **R4** (`S360-310-R4`). It exists to
close the documentation gap recorded at
[`docs/hardware/hardware-artifact-policy.md` Follow-up PR sequence #3 (archived)](../archive-index.md)
and at
[`docs/hardware/board-readiness-matrix.md` Follow-up PR sequence #1 (archived)](../archive-index.md),
which together call for a per-board pin/package mapping audit
**before** any `FanRelay`-bearing package, product, build, or
WebFlash work can proceed.

When HW-PINMAP-310 originally landed, the audit was
`pending — schematic/design evidence required`: no module-side
`S360-310-R4` schematic existed in this repository, and no curated
artifact index built from it had been committed. **HW-ASSETS-310 has
since landed** — the module-side schematic PDF is committed at
[`docs/hardware/schematics/S360-310-R4.pdf`](schematics/S360-310-R4.pdf)
and the curated artifact index lives at
[`docs/hardware/artifacts/S360-310-R4.md`](artifacts/S360-310-R4.md).
**HW-PINMAP-310-FOLLOWUP** (this revision) consumes that evidence to
record a schematic-backed reconciliation view. It does **not**
resolve the firmware-side disagreements it surfaces — those are
owed to separate, evidence-bearing follow-up PRs (silkscreen /
harness / BOM / bench evidence; `PACKAGE-RELAY-001`;
`CORE-ABSTRACT-BUS-001`). The bench / silkscreen / harness / BOM /
load-contact evidence-capture **checklist** is recorded under
`S360-310-BENCH-001` at
§[S360-310-BENCH-001 — Relay bench evidence](#s360-310-bench-001--relay-bench-evidence);
every one of the ten enumerated hardware-evidence rows currently
sits at `pending — bench evidence required`.

The schematic-backed evidence on file proves the **logical**
module-side `J2` ↔ Core-side `J4` net match (`+5V` / `Relay` /
`GND`) and the module-side relay drive topology (`Q1` MMBT3904
NPN low-side, `R1` 1 kΩ base, `R2` 10 kΩ pull-down, `D1` flyback,
`K1` mechanical relay, `J1` 3-pin "Inline Fan" load-side). It does
**not** prove silkscreen pin-1 orientation on either side, does
**not** prove `J1` NO / COM / NC mapping, does **not** prove the
`K1` part number or coil-voltage / contact-current rating, does
**not** establish the Core-to-module harness identity, and does
**not** resolve the systemic `IO3` (Core schematic) vs `GPIO4`
(ceiling Core abstract `relay_pin`) vs `GPIO10` (generic Core
abstract `relay_pin`) disagreement — that resolution remains owned
by `CORE-ABSTRACT-BUS-001` per
[`release-one-hardware-audit.md` Required follow-ups #2 / #3 (archived)](../archive-index.md),
not by HW-PINMAP-310 or its FOLLOWUP.

## Schematic summary

This section restates only what is directly visible in the
committed schematic PDF

- **Section title: "INLINE FAN CONTROL USING RELAY".** One
  section, no sub-sheets. Single-sheet A4 KiCad 10.0.3 export;
  title-block File field reads `S360-310-R4.kicad_sch`; Title /
  Date / Rev fields blank.
- **"From Core" connector `J2`** — 3-pin. Net list as labelled in
  the PDF:

  | Module-side `J2` pin | Net |
  |---:|---|
  | 1 | `+5V` |
  | 2 | `Relay` |
  | 3 | `GND` |

  The net order (`+5V` → `Relay` → `GND`) matches the Core-side
  `J4` capture in
  [`s360-100-r4-core.md` §J4](s360-100-r4-core.md#j4--relay-module-connector-3-pin)
  pin-for-pin by net.
- **Relay coil-drive topology.** Low-side NPN BJT switch from a
  module-supplied `+5V` rail (no on-board boost):
  - `Q1 MMBT3904` — NPN BJT in SOT-23. Emitter to `GND`,
    collector to the relay coil node.
  - `R1 1 kΩ` — base series resistor on the `Relay` signal path
    from `J2` pin 2 to the `Q1` base node.
  - `R2 10 kΩ` — base-to-`GND` pull-down at the `Q1` base node
    (keeps `Q1` off when the `Relay` net is floating or Hi-Z).
  - `D1` — flyback diode across the relay coil (cathode to `+5V`,
    anode to the `Q1` collector / coil node). No part number
    visible.
- **Relay `K1`.** Mechanical relay. Coil between `+5V` and `Q1`
  collector; flyback `D1` across the coil as above. **No relay
  part number, no coil-voltage label, and no contact-current
  rating visible on the schematic sheet.** The relay reference is
  `K1` only.
- **"Inline Fan" load-side connector `J1`** — 3-pin. Wired to the
  switching-contact side of `K1` (visible contact-side pin labels
  `1` / `2` / `3`). A 3-pin output is consistent with SPDT contact
  topology (`NO` / `COM` / `NC`), but the specific NO / COM / NC
  assignment to `J1` pins 1 / 2 / 3 is **not** labelled on the
  schematic.
- **What the schematic does NOT visibly contain.**
  - No opto-isolator between the `Relay` input and the relay coil
    (the `Q1` MMBT3904 is a direct low-side switch).
  - No on-board indicator LED.
  - No mains-side snubber across the `K1` switching contacts.
  - No "NEXTION DISPLAY" connector (unlike `S360-311-R4` /
    `S360-312-R4`).
  - No mounting-hole references (`H1`..`H4`).
  - No I²C / UART / DAC / boost-converter circuitry.

Everything beyond the list above — module dimensions, exact
component values not printed on the visible sheet, on-board PCB
silkscreen, layer stackup, fab notes, harness identity, or any
electrical timing / contact-rating / isolation-rating claim — is
**unknown** and would have to come from the

## Relay drive findings

The module-side `Relay` net (at `J2` pin 2) feeds the base of `Q1`
through `R1` (1 kΩ series); `R2` (10 kΩ) pulls the base to `GND`;
`Q1` (MMBT3904 NPN) sinks the relay coil; `D1` is the flyback
diode across the coil; the coil's high side is the module-supplied
`+5V` rail at `J2` pin 1.

| Element | Source | Value | Status |
|---|---|---|---|
| Drive transistor | `S360-310-R4.pdf` | `Q1` MMBT3904 NPN BJT (SOT-23) | Schematic-confirmed. |
| Base series resistor | `S360-310-R4.pdf` | `R1` 1 kΩ | Schematic-confirmed. |
| Base pull-down | `S360-310-R4.pdf` | `R2` 10 kΩ to `GND` | Schematic-confirmed. |
| Flyback diode | `S360-310-R4.pdf` | `D1` across the relay coil (cathode to `+5V`, anode to `Q1` collector) | Schematic-confirmed; **part number not visible**. |
| Coil rail | `S360-310-R4.pdf` | Module-supplied `+5V` at `J2` pin 1; no on-board boost | Schematic-confirmed. |
| Drive-level compatibility with 3.3 V ESP32 GPIO | Topology with 1 kΩ base series + 10 kΩ pull-down | **Consistent with** direct 3.3 V CMOS drive into MMBT3904 base; no level shifter is required between the ESP32 and the module input | Schematic-backed at the topology layer **only**. Does **not** establish which Core-side GPIO physically carries the `Relay` net — see [§Reconciliation findings](#reconciliation-findings) and [§Existing package abstraction](#existing-package-abstraction). |
| Coil current vs Q1 SOA | `S360-310-R4.pdf` shows no coil-current label | Unknown | **Pending — `K1` BOM identity required** (MMBT3904 max Ic ≈ 200 mA; whether the chosen `K1` coil falls inside SOA depends on `K1`'s coil current draw). |
