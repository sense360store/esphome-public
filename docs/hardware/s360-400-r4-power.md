# S360-400-R4 240v PSU Pin / Package Mapping Audit (HW-PINMAP-400)

## Status

**Status: partial — schematic evidence available; package
reconciliation, BOM, silkscreen, creepage/clearance, and
COMPLIANCE-001 pending.**

This document is the **HW-PINMAP-400** audit record for the
`Sense360 240v PSU` board, revision **R4** (`S360-400-R4`). It exists
to close the documentation gap recorded at
[`docs/hardware/hardware-artifact-policy.md` Follow-up PR sequence #7 (archived)](../archive-index.md)
and at
[`docs/hardware/board-readiness-matrix.md` Follow-up PR sequence #5 (archived)](../archive-index.md),
which together call for a per-board pin/package mapping audit
**before** any `PWR`-bearing package, product, build, or WebFlash
work can proceed, and **additionally** require the
**COMPLIANCE-001 mains-voltage UK / EU sign-off** before the board
may be promoted on any product surface.

When HW-PINMAP-400 originally landed, the audit was
`pending — schematic/design evidence required`: no module-side
`S360-400-R4` schematic existed in this repository, and no curated
artifact index built from it had been committed. **HW-ASSETS-400
has since landed as PR #514** — the module-side schematic PDF is
committed at
[`docs/hardware/schematics/S360-400-R4.pdf`](schematics/S360-400-R4.pdf)
(461,206 bytes; SHA256
`295e3ec9192603fd4ca7d89b8cda68777e5cb8e9713ed8b0fba2316babb0e765`;
byte-identical to the upload) and the curated artifact index lives
at
[`docs/hardware/artifacts/S360-400-R4.md`](artifacts/S360-400-R4.md).
**HW-PINMAP-400-FOLLOWUP** (this revision) consumes that evidence
to record a schematic-backed reconciliation view. It does **not**
resolve the part-identity / silkscreen / BOM / creepage / clearance
/ compliance disagreements it surfaces — those are owed to
separate, evidence-bearing follow-up PRs (BOM cross-check;
silkscreen photographs or KiCad PCB source; bench / load / thermal
/ EMI evidence; `PACKAGE-POWER-400-001`; `COMPLIANCE-001`).

The schematic-backed evidence on file proves the **logical**
mains-input topology (`J1` 3-pin AC input `LIVE` / `NEUTRAL` /
`Earth_Protective`; `F1` resettable fuse on the LIVE leg; `RV1`
MOV across the AC line; `C1` X-cap across the AC line), the
**AC/DC converter reference** (`PS1` value `HLK-10M05` with
pinout `AC(L)` / `AC(N)` / `-VO` / `+VO`), the **output-filter
topology** (four-cap bank `C5` / `C6` / `C7` / `C8` between
`+VO` and `-VO`/`GND`), the **output connector** (`J2` 2-pin
`+5VP` / `GND`), and the **absence** of Y-class capacitors,
common-mode / differential-mode line-filter inductors, secondary
regulators / boost / LDO, on-board indicator LED, thermal cutout
or NTC inrush limiter, and any I²C / UART / SPI / GPIO circuitry.

The schematic does **not** prove silkscreen pin-1 orientation on
either `J1` or `J2`, does **not** prove the physical connector
identity / current rating of either `J1` (mains-rated 3-pin) or
`J2` (low-voltage 2-pin), does **not** establish the per-component
ratings implied by the value-field strings (`F1 A250-1200` hold /
trip / voltage; `RV1 10D391K` clamp voltage / energy; `C1 470nF`
X-cap safety class X1 / X2; output-filter cap voltage / dielectric
/ ESR), does **not** prove creepage / clearance / isolation /
earth-bonding / thermal / inrush / EMI / EMC behaviour, and does
**not** resolve the three-way AC/DC part-identity disagreement
between the catalog
([`config/hardware-catalog.json`](../../config/hardware-catalog.json)
line 109 `Mains to 5V using HLK-5M05.`), the package header
([`packages/hardware/power_240v.yaml`](../../packages/hardware/power_240v.yaml)
line 7 `HLK-PM01 or similar`), and the schematic (`PS1 = HLK-10M05`).
That resolution is owed to BOM evidence under
`PACKAGE-POWER-400-001`. The mains-voltage UK / EU sign-off is
owed to `COMPLIANCE-001` and is **not** in scope for this PR; a
schematic is not a compliance artefact.

schematic does **not** enumerate a mating connector for `S360-400`
