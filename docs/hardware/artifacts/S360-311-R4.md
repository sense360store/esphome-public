# S360-311-R4 Hardware Artifact Index (HW-ASSETS-003)

## Purpose and scope

This document is the **curated artifact index** for the Sense360 PWM
fan board, revision R4 (`S360-311-R4`). It records the schematic
evidence delivered under HW-ASSETS-003, classifies it against the
[Hardware Artifact Policy (archived)](../../archive-index.md), and
inventories what is still missing before pin-map, package, product,
or WebFlash work can proceed.

This index is **documentation only**. It does not:

- change any value in [`config/hardware-catalog.json`](../../../config/hardware-catalog.json),
  [`config/product-catalog.json`](../../../config/product-catalog.json),
  [`config/webflash-builds.json`](../../../config/webflash-builds.json),
  or [`config/webflash-compatibility.json`](../../../config/webflash-compatibility.json),
- change any product YAML under [`products/`](../../../products/),
  any WebFlash wrapper under
  [`products/webflash/`](../../../products/webflash/), or any package
  YAML under [`packages/`](../../../packages/),
- change any workflow under `.github/workflows/`, any script under
  `scripts/`, any test under `tests/`, any component under
  `components/`, any header under `include/`, or `.gitignore`,
- mark `S360-311-R4` `verified` — `schematic_status` stays
  `cataloged_unverified`,
- add or change `schematic_file` for `S360-311` in
  [`config/hardware-catalog.json`](../../../config/hardware-catalog.json),
- promote `S360-311` to `preview` / `stable` / `production`,
- add a `FanPWM`-bearing entry to
  [`config/webflash-builds.json`](../../../config/webflash-builds.json)
  or to any product / WebFlash wrapper,
- regenerate firmware, create a release, or change any WebFlash
  manifest.

**Schematic evidence uploaded ≠ schematic verified.** Promotion from
`cataloged_unverified` to `verified` requires the `HW-PINMAP-311`
follow-up (pin-map reconciliation against the silkscreen, the Core J6
1-to-13 verify flag, and the package YAML
[`packages/expansions/fan_pwm.yaml`](../../../packages/expansions/fan_pwm.yaml))
plus a separate JSON catalog change.

The Release-One shipping configuration remains
`Ceiling-POE-VentIQ-RoomIQ` with artifact
`Sense360-Ceiling-POE-VentIQ-RoomIQ-v1.0.0-stable.bin`. The LED
preview path (`Ceiling-POE-VentIQ-RoomIQ-LED`) is unchanged.
FanPWM is not, and is not made by this index, part of any current
Release-One or WebFlash build.

### What the schematic appears to contain

Observed directly from the uploaded PDF. Every claim below is sourced
from a visible label or symbol on the single schematic sheet.

- **Voltage Boost** section. `MT3608` boost converter (`U1`), input
  inductor `L1 22µH`, Schottky `D1 SS34`, feedback divider `R3 38k`
  / `R5 2k`, input cap `C1 22µF/10V`, output cap `C2 22µF/10V`,
  output indicator `R4 330R` + blue LED `D2`. Boosts the Core-supplied
  `+5V` rail up to `+12V` for the fan outputs.
- **"NINE 4pin FANs"** section. Section title as printed in the PDF
  (see [Open questions](#open-questions--verification-needed) #2).
  **Four** 4-pin fan output connectors are present: `J1`, `J2`, `J4`,
  `J5`. Each fan connector has pins (`+12V`, `GND`, `Pul_Cou*`,
  `TachPMW*`) and is paired with:
  - one low-side N-channel MOSFET (`Q_NMOS_GSD` — `Q1..Q4`) on the
    `TachPMW*` gate path, with a 1 kΩ gate resistor (`R1`, `R2`, `R6`,
    `R7`),
  - drain wired to the shared `TachIO` net,
  - source to `GND`.
- **"From Core"** connector `J3` — 13-pin. Net list as labelled in
  the PDF:

  | Module J3 pin | Net |
  |---:|---|
  | 1 | `+5V` |
  | 2 | `TachIO` |
  | 3 | `TachPMW1` |
  | 4 | `Pul_Cou1` |
  | 5 | `TachPMW2` |
  | 6 | `Pul_Cou2` |
  | 7 | `TachPMW3` |
  | 8 | `Pul_Cou3` |
  | 9 | `TachPMW4` |
  | 10 | `Pul_Cou4` |
  | 11 | `UART_RX` |
  | 12 | `UART_TX` |
  | 13 | `GND` |

- **"NEXTION DISPLAY"** connector `J6` — 4-pin: `+5V`,
  `ESP32_TXD` / `UART_TX` (to screen RX), `ESP32_RXD` / `UART_RX`
  (to screen TX), `GND`.
- **Mounting holes** `H1`, `H2`, `H3`, `H4`.

### Reconciliation flags raised or strengthened by this schematic

1. **Core J6 1-to-13 pin order (pre-existing **verify** flag).**
   [`s360-100-r4-core.md` §J6](../s360-100-r4-core.md#j6--12-v-pwm-fan-connector-13-pin)
   marks the 1-to-13 pin order on the Core-side J6 as **verify** against
   silkscreen. The module-side `J3` ordering recorded above
   (`+5V` → `TachIO` → `TachPMW1` → `Pul_Cou1` → … → `Pul_Cou4` →
   `UART_RX` → `UART_TX` → `GND`) is schematic-only until physical
   silkscreen evidence resolves the verify flag. Belongs to
   `HW-PINMAP-311`.
2. **UART pins on the 13-pin Core ↔ Module harness (new flag).** The
   module-side `J3` carries `UART_RX` on pin 11 and `UART_TX` on pin 12
   — these are then routed on-board to the Nextion display connector
   `J6`. The Core-side `J6` capture in
   [`s360-100-r4-core.md` §J6](../s360-100-r4-core.md#j6--12-v-pwm-fan-connector-13-pin)
   does **not** currently document UART pins on the 13-pin connector.
   Two possibilities, both in scope for `HW-PINMAP-311`:
   - the Core J6 capture is incomplete and must be extended to cover
     pins 11 / 12, or
   - the Nextion UART pair is delivered to the module via a separate
     harness, not via the 13-pin Core J6 cable.

   This index records the discrepancy; it does **not** edit
   [`s360-100-r4-core.md`](../s360-100-r4-core.md) or
   [`packages/expansions/fan_pwm.yaml`](../../../packages/expansions/fan_pwm.yaml).
3. **`TachPMW*` / `Pul_Cou*` source pins (pre-existing, unchanged).**
   On the Core side, `TachPMW1..4` and `Pul_Cou1..4` are driven by the
   SX1509 expander
   ([`s360-100-r4-core.md` Fan / driver outputs](../s360-100-r4-core.md#fan--driver-outputs);
   [`packages/expansions/gpio_expander_sx1509.yaml`](../../../packages/expansions/gpio_expander_sx1509.yaml)
   channels 0–3 fan PWM, 4–7 tach). `TachIO` is the direct ESP32 `IO16`
   passthrough. The module schematic does not change this; it confirms
   the per-fan low-side N-FET TachPMW gating topology on the module
   side.
4. **Output rail and fan count.** The module boosts the Core-supplied
   `+5V` to `+12V` on board (MT3608) and exposes four 4-pin fan
   outputs. This matches the hardware-catalog description
   "12V PWM fan driver, up to 4 fans with tach feedback." It does
   **not** match the section title `"NINE 4pin FANs"` printed inside
   the schematic — see [Open questions](#open-questions--verification-needed)
   #2.

This index does not edit
[`docs/hardware/firmware-package-mapping-audit.md` (archived)](../../archive-index.md)
or any package YAML.

## Open questions / verification needed

Items below are the **artifact-specific** opens raised by this index.
Pre-existing pin-map open questions are tracked authoritatively in
[`s360-100-r4-core.md` Open questions](../s360-100-r4-core.md#open-questions--verification-needed),
[`board-readiness-matrix.md` §S360-311 (archived)](../../archive-index.md),
and
[`remaining-board-documentation-audit.md` §Sense360 PWM (archived)](../../archive-index.md);
they are not re-litigated here.

1. **`HW-PINMAP-311` — pin / package mapping audit.** Reconcile the
   module-side `J3` against the Core-side `J6` (including the 1-to-13
   verify flag) and against
   [`packages/expansions/fan_pwm.yaml`](../../../packages/expansions/fan_pwm.yaml).
   Pre-requisite for any FanPWM productization.
2. **`"NINE 4pin FANs"` section title.** The schematic's section
   title reads `"NINE 4pin FANs"` but the sheet shows **four** 4-pin
   fan output connectors (`J1`, `J2`, `J4`, `J5`). The
   hardware-catalog description (`"up to 4 fans"`) matches the visible
   four connectors; the `"NINE"` label is unexplained. Whether this is
   a stale label copied from a different design, a marker / project
   code, or an undelivered intent is unresolved. **Documentation
   question only**; not a wiring change.
3. **UART pins on `J3` 11 / 12.** Whether the Nextion UART pair is
   delivered to the module via the 13-pin Core `J6` cable (in which
   case the Core J6 capture is incomplete) or via a separate harness
   (in which case the module-side `J3` 11 / 12 UART labels refer to a
   different connector populated elsewhere on the harness). Resolution
   in `HW-PINMAP-311`.
4. **Long-term storage for retained-but-not-committed artifacts.**
   Same open as for S360-100-R4 — see
   [`docs/hardware/artifacts/S360-100-R4.md` Open questions #1 / #5](./S360-100-R4.md#open-questions--verification-needed).
5. **Standalone hardware reference doc.** No `s360-311-r4-*.md`
   exists yet. `HW-PINMAP-311` should produce one.
