# S360-200-R4 Hardware Artifact Index (HW-BOM-ASSETS-001 — partial)

## Purpose and scope

This document is the **curated artifact index** for the Sense360
RoomIQ room sensor board, revision R4 (`S360-200-R4`). It records
the BOM evidence delivered under `HW-BOM-ASSETS-001` and classifies
it against the
[Hardware Artifact Policy (archived)](../../archive-index.md). It is
the first per-board artifact index for `S360-200-R4`; prior to this
PR the board was tracked as `Artifact index: missing
(HW-ASSETS-003 deferred)` in
[`docs/hardware/board-readiness-matrix.md` (archived)](../../archive-index.md).

This index is **documentation only**. It does not:

- change any value in [`config/hardware-catalog.json`](../../../config/hardware-catalog.json),
  [`config/product-catalog.json`](../../../config/product-catalog.json),
  [`config/webflash-builds.json`](../../../config/webflash-builds.json),
  or [`config/webflash-compatibility.json`](../../../config/webflash-compatibility.json),
- change any product YAML under [`products/`](../../../products/),
  any WebFlash wrapper under [`products/webflash/`](../../../products/webflash/),
  or any package YAML under [`packages/`](../../../packages/),
- change any workflow under `.github/workflows/`, any script under
  `scripts/`, any test under `tests/`, any component under
  `components/`, any header under `include/`, or `.gitignore`,
- commit the BOM spreadsheet itself to the repository — per the
  current [Hardware Artifact Policy (archived)](../../archive-index.md)
  (HW-ASSETS-001), `*_BOM.xlsx` files are **retained-but-not-committed**
  and recorded by size + SHA256 only,
- promote `S360-200` away from its current Release-One posture
  (`schematic_status: verified`; consumed by
  `Ceiling-POE-VentIQ-RoomIQ` and `Ceiling-POE-VentIQ-RoomIQ-LED`),
- resolve the Core J10 vs RoomIQ J6 12-pin pin-order discrepancy
  (still tracked under
  [`docs/hardware/firmware-package-mapping-audit.md` Core J10 vs RoomIQ J6 (archived)](../../archive-index.md)
  as `needs-silkscreen/bench-verification`),
- close any open question listed in
  [`docs/hardware/s360-200-r4-roomiq.md` Open questions](../s360-200-r4-roomiq.md#open-questions--verification-needed).

**BOM evidence available ≠ board reconciliation complete.** This
index records what the BOM says; reconciling the BOM against the
schematic, against the per-board reference doc, and against the
package YAMLs that consume this board is owed to a separate
follow-up PR.

## Policy reference

This index applies
[`docs/hardware/hardware-artifact-policy.md` (archived)](../../archive-index.md)
(HW-ASSETS-001). The relevant rules:

- BOM spreadsheets are **retained-but-not-committed** under the
  current per-board decision. Filename, size, and SHA256 are
  recorded here; the `.xlsx` itself is not added to `git`.
- Per-board indexes record file sizes and SHA256 for every artifact
  actually present in the upload. Missing files are marked
  `not provided in this upload`; checksums are never fabricated.
- Hardware artifact availability does **not** imply firmware
  support, WebFlash availability, build-matrix inclusion,
  product-catalog status, or Release-One promotion.
- No Git LFS is introduced. A long-term storage decision for
  retained-but-not-committed artifacts is **open**.

### BOM-confirmed evidence for downstream reconciliation

- **On-board sensor identity confirmed:** `LTR-303ALS-01` (ALS),
  `SHT45-AD1B-R3` (SHT4x family), `EKMC1601111` (PIR), `BMP581`
  (pressure). This matches the main-components list in
  [`s360-200-r4-roomiq.md` Main components](../s360-200-r4-roomiq.md#main-components).
- **Module-connector identity confirmed:** `J6` is a JST SH
  `SM12B-SRSS-TB` 12-pin horizontal connector. This matches the
  12-pin RoomIQ-to-Core connector described in
  [`s360-200-r4-roomiq.md` Module connector](../s360-200-r4-roomiq.md#module-connector).
- **Off-board radar connectors `J2` / `J3`:** the BOM lists `J2`
  as a 2×4 (8-pin) pin socket on 2.00 mm pitch and `J3` as a 1×5
  pin socket on 2.54 mm pitch. The standalone reference doc
  describes `J2` as the 8-pin LD2450 mmWave radar connector and
  `J3` as the 5-pin DFRobot SEN0609 / C4001 radar connector;
  this BOM is consistent with that role but does **not** by
  itself prove pin assignment — silkscreen / bench reconciliation
  is still owed.
- **Pull-up biasing:** `R1`, `R2`, `R3` are all 10 kΩ. The
  hardware reference doc records two 10 kΩ resistors near `U1`
  (one of which pulls `ALS_INT` to +3.3 V — exact assignment
  marked `verify`) plus one near the BMP581 `add` net at `JP1`.
  The BOM line items confirm the value and quantity but do not
  resolve the `verify` flag on which net each pulls.
- **`JP1` jumper:** the standalone reference doc records a
  3-pin jumper `JP1` for BMP581 I²C-address selection. The BOM
  does **not** list a `JP1` reference. This may be a
  silkscreen-only / footprint-only feature, a populated solder
  jumper on `U4`'s `SDO/SA0` net, or a BOM omission; reconciliation
  is owed to a follow-up.

The BOM is sparse on decoupling-capacitor placement detail beyond
the references listed; cross-reference with the schematic PDF
remains the source of truth for which net each capacitor
decouples.

## Open questions / verification needed

The HW-BOM-ASSETS-001 BOM ingest does **not** by itself close any
of the open questions in
[`s360-200-r4-roomiq.md` Open questions](../s360-200-r4-roomiq.md#open-questions--verification-needed)
or the Core J10 vs RoomIQ J6 pin-order discrepancy tracked in
[`firmware-package-mapping-audit.md` (archived)](../../archive-index.md).
The following items remain open after this index lands:

1. **Core `J10` vs RoomIQ `J6` 12-pin pin-order alignment.** The
   BOM confirms `J6` is a 12-pin `SM12B-SRSS-TB` JST connector,
   but the 1-to-12 pin-order alignment between Core `J10` and
   RoomIQ `J6` still needs silkscreen / bench verification.
2. **`JP1` jumper presence / function.** Whether the BMP581
   I²C-address-selection jumper described in
   [`s360-200-r4-roomiq.md`](../s360-200-r4-roomiq.md) is
   silkscreen-only, a solder-jumper, or a missing BOM line.
3. **`R1` / `R2` / `R4` exact roles.** The BOM lists three 10 kΩ
   resistors (`R1`, `R2`, `R3`) of footprint `R_0603_1608Metric`;
   the standalone reference doc additionally mentions an `R4`
   near `U3` on the PIR output side with a `verify` flag on pull
   direction. The BOM has no `R4` entry. Whether `R4` is a
   schematic-only reference, a depopulated position, or a BOM
   omission needs follow-up.
4. **`AirQ_Led` / `AirQ_Status_Led` reuse.** This board is on the
   RoomIQ side of HW-002 Open Question #4; the BOM does not
   resolve that question.
5. **CPL / silkscreen / harness / bench evidence.** No CPL, no
   gerbers, no STEP, no board images, no bench evidence
   accompanied this BOM upload.
