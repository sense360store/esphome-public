## Purpose and scope

PSU board, revision R4 (`S360-410-R4`). It records the schematic
evidence delivered under HW-ASSETS-410, classifies it against the
[Hardware Artifact Policy (archived)](../../archive-index.md), and
inventories what is still missing before any pin-map, package,
product, WebFlash, release, or import work can proceed.

## Policy reference

This index applies
[`docs/hardware/hardware-artifact-policy.md` (archived)](../../archive-index.md)
(HW-ASSETS-001). Relevant rules:

- Curated schematic PDFs may be committed directly. The S360-410-R4
  schematic PDF is committed by this PR at
  [`docs/hardware/schematics/S360-410-R4.pdf`](../schematics/S360-410-R4.pdf).
- Mac OS metadata, editor / IDE history, embedded `.git/`, vendor
  scratch exports, KiCad temporary / cache / backup files, and
  private supplier / customer data are excluded outright.
- KiCad source, KiCad PCB, KiCad project metadata, BOM, CPL,
  gerbers, drill files, STEP, and board images are
  **retained-but-not-committed** under the current policy.
- Per-board indexes record file sizes and SHA256 for every artifact
  actually present in the upload. Missing files are marked
  `not provided in this upload`; checksums are never fabricated.
- **Hardware artifact availability does not imply firmware support,
  WebFlash availability, build-matrix inclusion, product-catalog
  status, IEEE 802.3af / 802.3at compliance, isolation / safety
  certification, thermal / EMI / EMC qualification, or Release-One
  promotion.**
- A schematic alone is not sufficient evidence to ship a PoE supply.
  Bench / load / link-up / isolation / thermal / EMI evidence is a
  separate evidence class.

### What the schematic appears to contain

Observed directly from the uploaded PDF. Every claim below is sourced
from a visible label, value, or symbol on the single schematic sheet.

- **Two visible section labels:** `"PoE Power Supply"` (upper block,
  covering the LAN connector + PoE PD controller + buck stage) and
  `"Galvanic Isolated Part"` (lower dashed boundary, covering the
  pre-isolation network up to and including the isolated DC/DC
  module). The dashed boundary marks the galvanic-isolation domain
  on the sheet.
- **`LAN_CON1` integrated magnetics / RJ45 connector module** — part
  field reads `RJP-003TC1(LPJ4112CNL)`, schematic legend "Single
  10/100 BASE-TX Filter Connector Module". Pin map as labelled on
  the schematic:

  | `LAN_CON1` pin | Net (as labelled) |
  |---:|---|
  | 0 | `Shield` |
  | 1 | `TD+` |
  | 2 | `TD-` |
  | 3 | `RD+` |
  | 4 | `TCT` |
  | 5 | `RCT` |
  | 6 | `RD-` |
  | 7 | `GND/NC` |
  | 8 | `NC/GND` |
  | 9 | `PW+` |
  | 10 | `PW-` |

  Adjacent labels on the schematic name the magnetics-side pin pairs
  `J1`, `J2`, `J3`, `J6` (data-pair termini), `J4,5` (centre-tap
  network), and `J7,8` (PoE power-out side). Bob-Smith / mid-span
  termination components labelled adjacent to the centre-tap network
  are `2x1000pF/2KV` (bridge caps) and `2x75 Ohm` (terminations).
  `C3 1nF` is connected between the `Lan_earth` net and the
  `Spare1` / `Spare2` rails.
- **PoE PD controller `U1`** — part field reads
  `TPS2378DDAR(HSOIC-8)`. Pin map as labelled on the schematic:

  Adjacent annotations on the sheet record:

    the schematic note "Connect 24.9 kΩ to VDD for detection."
  - `R2` value `1.27k` — classification programming resistor on
    `CLS`, with the schematic-visible label `Class=0 (0.44 to 12.95W)`.
  - `D1` part field reads `SMAJ58A` — TVS diode across the PoE input.
  - `C1` value `100n` — local decoupling.
  - `C2` value `15uf`, labelled `CBULK` on the sheet.
  - `R5` value `0.03R` — current-sense / `RTN` shunt.
  - Schematic notes on the sheet:
    - "Connect resistor from CLS to VSS to program classification
      current. The controller places a voltage of approximately 2.5 V
      across the external resistor whenever the voltage differential
      between VDD and VSS lies from about 10.9 V to 22 V. Connect

    - "Active low, open-drain converter disable output, referenced to
      RTN."
    - "Active low indicates type 2 PSE connected or APD active. If
      not used, connect APD to RTN."
    - "Auxiliary Power Detect" (next to `APD`).
    - "Drain of PoE pass MOSFET."
    - "Rmps is for to Maintain Power Signature (MPS). A valid MPS
      consists of a minimum dc current of 10 mA and an AC impedance
      lower than 26.3 k in parallel with 0.05 uF. The AC impedance is
      usually accomplished by the minimum operating CBULK requirement
      of 5 uF."

## Safety / PoE note

The committed schematic is **schematic evidence**, not **compliance
evidence**. It does **not**:

- make any **IEEE 802.3af / 802.3at compliance** claim. The
  schematic-visible `Class=0 (0.44 to 12.95W)` annotation declares
  the intended PD classification on the design; it is **not** a
  compliance test against a PSE,
- make any **isolation rating** claim. The schematic shows an
  `F0505S-2WR2(SIP-7)` (with `AM1D-0505S-NZ` annotated alternate)
  isolated DC/DC module providing the galvanic-isolation barrier
  between the post-PoE-controller buck output `Sw_Vin_Poe` and the
  Core-facing `+5VP` / `GND` output, but the isolation rating is a
  BOM / datasheet claim and the as-built insulation resistance /
  Hi-pot / leakage is bench evidence,
- make any **earth-continuity** claim. The schematic shows four
  mounting holes `H1`–`H4` each labelled `Earth`, but PCB-level
  bonding to the RJ45 shield through plated holes / pads is **not**
  derivable from the schematic alone,
- make any **thermal** claim. Continuous-load thermal rise of `U1`
  (TPS2378), `U2` (TX4138), and `DCDC1` (F0505S-2WR2) is **not**
  schematic-derivable,
- make any **EMI / EMC** claim. Conducted / radiated emissions and
  immunity are **not** schematic-derivable,
- make any **PoE link-up / bench load** claim. Observed link-up
  against an 802.3af or 802.3at PSE, observed signature /
  classification behaviour, observed load regulation across the
  rated 5 V output range, and observed cold-start inrush are all
  bench evidence and are **not** in this upload,
- imply approval for **standard** WebFlash exposure beyond the
  pre-existing Release-One consumption under the preserved

- imply approval for any **new** PoE-410-explicit product entry —
