# S360-210-R4 Hardware Artifact Index (HW-BOM-ASSETS-001 — partial)

## Purpose and scope

This document is the **curated artifact index** for the Sense360
AirIQ air-quality ceiling sensor board, revision R4
(`S360-210-R4`). It records the BOM evidence delivered under
`HW-BOM-ASSETS-001` and classifies it against the
[Hardware Artifact Policy (archived)](../../archive-index.md). It is
the first per-board artifact index for `S360-210-R4`; prior to
this PR the board was tracked as `Artifact index: missing
(HW-ASSETS-004 deferred)` in
[`docs/hardware/board-readiness-matrix.md` (archived)](../../archive-index.md).

This index is **documentation only**. It does not:

- change any value in [`config/hardware-catalog.json`](../../../config/hardware-catalog.json),
  [`config/product-catalog.json`](../../../config/product-catalog.json),
  [`config/webflash-builds.json`](../../../config/webflash-builds.json),
  or [`config/webflash-compatibility.json`](../../../config/webflash-compatibility.json),
- change any product YAML under [`products/`](../../../products/),
  any WebFlash wrapper under [`products/webflash/`](../../../products/webflash/),
  or any package YAML under [`packages/`](../../../packages/),
- change any workflow under `.github/workflows/`, any script under
  `scripts/`, any test under `tests/`, any component under
  `components/`, any header under `include/`, or `.gitignore`,
- commit the BOM spreadsheet itself — per the current
  [Hardware Artifact Policy (archived)](../../archive-index.md),
  `*_BOM.xlsx` files are **retained-but-not-committed** and
  recorded by size + SHA256 only,
- promote `S360-210` away from its current posture
  (`schematic_status: verified`; **mutually exclusive** with
  `VentIQ` per
  [`config/webflash-compatibility.json`](../../../config/webflash-compatibility.json)
  `rules.airiq_and_ventiq_mutually_exclusive: true`; **no active
  product YAML consumes AirIQ today** — Release-One ships VentIQ),
- add `AirIQ` to any product YAML, WebFlash wrapper, build matrix,
  config string, or `artifact_name`,
- resolve HW-002 Open Question #4 (`AirQ_Led` / `AirQ_Status_Led`
  reuse on AirIQ vs VentIQ).

**BOM evidence available ≠ AirIQ WebFlash-shippable.** This
index records what the BOM says; productizing an AirIQ-bearing
config is a separate set of gates (product-catalog entry,
build-matrix entry, onboarding-gate clearance, mutex
re-evaluation with VentIQ) and is **not** unblocked here.

## Policy reference

This index applies
[`docs/hardware/hardware-artifact-policy.md` (archived)](../../archive-index.md)
(HW-ASSETS-001). The relevant rules:

- BOM spreadsheets are **retained-but-not-committed** under the
  current per-board decision. Filename, size, and SHA256 are
  recorded here; the `.xlsx` itself is not added to `git`.
- Per-board indexes record file sizes and SHA256 for every artifact
  actually present in the upload. Missing files are marked
  `not provided in this upload`; checksums are never fabricated.
- Hardware artifact availability does **not** imply firmware
  support, WebFlash availability, build-matrix inclusion,
  product-catalog status, or Release-One promotion.
- No Git LFS is introduced.

### BOM-confirmed evidence for downstream reconciliation

- **On-board sensor identity confirmed:** `SGP41-D-R4` (VOC /
  NOₓ), `SCD41-D-R2` (CO₂), `MICS-4514` (multi-gas), and
  `STM8S003F3U6TR` (MICS-to-I²C bridge MCU). All four match
  the [`s360-210-r4-airiq.md` Main components](../s360-210-r4-airiq.md#main-components-visible-on-this-sheet)
  list.
- **`SFA40-D-Rx` formaldehyde sensor is on-board.** This is a
  **clarification against the standalone reference doc**, which
  describes "**SFA40** connector — formaldehyde (HCHO) sensor
  connector. Sensor lives off-board on a separate Sensirion
  module." The BOM lists `U2 = SFA40-D-Rx` with the
  `SFA40-D-Rx:SFA40-D-R` footprint, indicating the chip is
  populated directly on the board rather than reached through an
  off-board connector. Reconciliation of the standalone reference
  doc text is owed to a follow-up.
- **`LMV358` op-amp on-board.** Not enumerated in
  `s360-210-r4-airiq.md`. The op-amp is most likely tied to the
  MICS-4514 analog front-end signal conditioning that the STM8
  digitises. Reconciliation owed to a follow-up.
- **Status / indicator LEDs:** `D1` Blue (OSRAM) + `D2` Green
  (Lite-On). The `AirQ_Led` / `AirQ_Status_Led` reuse question
  (HW-002 Open Question #4) is **not** resolved by these BOM
  rows — which net drives which LED is silkscreen / schematic
  evidence, not BOM evidence.
- **Off-board sensor connectors:** `J2` (5-pin JST SH), `J3`
  (7-pin JST SH), `J4` (4-pin 2.00 mm pin header). The standalone
  reference doc lists `SFA40` and `SPS30` connectors but **does
  not** record connector pin-counts on the AirIQ-side; mapping
  each BOM `J*` to its sensor (SPS30 vs SFA40 vs Core `J9`
  module connector vs other) is owed to silkscreen / schematic
  reconciliation. The 7-pin `J3` is the strongest candidate for
  the Core-`J9` (AirIQ module connector) match given the Core
  side is 7-pin, but this PR does not claim that match without
  silkscreen evidence.
- **PoE / mains exposure:** none. `S360-210` is a low-voltage
  I²C / UART sensor module; no compliance exposure is introduced
  by this BOM.

The BOM is consistent with the standalone reference doc on the
core sensor set (SGP41, SCD41, MICS-4514 + STM8 bridge) and
diverges on two points (SFA40 on-board vs connector-only, and
the presence of the `LMV358` op-amp). Neither divergence
unblocks any downstream productisation work.

## Open questions / verification needed

The HW-BOM-ASSETS-001 BOM ingest does **not** by itself close any
of the open questions in
[`s360-210-r4-airiq.md`](../s360-210-r4-airiq.md) or
HW-002 Open Question #4. The following items remain open after
this index lands:

1. **`SFA40-D-Rx` on-board vs connector-only.** The standalone
   reference doc describes `SFA40` as a connector; the BOM lists
   `U2 = SFA40-D-Rx` as a populated chip. Reconciliation
   (schematic verification + doc-text fix) is owed.
2. **`LMV358` op-amp role.** Not described in the standalone
   reference doc. Likely tied to the MICS-4514 analog front
   end. Reconciliation owed.
3. **`J2` / `J3` / `J4` connector-to-sensor mapping.** Which
   `J*` is the Core `J9` module connector, which is the SPS30
   PM connector, and which is the SFA40-related connector
   (if any) needs silkscreen / schematic evidence.
4. **`AirQ_Led` / `AirQ_Status_Led` reuse (HW-002 Open
   Question #4).** Not resolved here. Which net drives `D1`
   Blue vs `D2` Green still needs silkscreen / schematic
   evidence.
5. **CPL / silkscreen / harness / bench evidence.** No CPL, no
   gerbers, no STEP, no board images, no bench evidence
   accompanied this BOM upload.
