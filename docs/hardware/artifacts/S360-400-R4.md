# S360-400-R4 Hardware Artifact Index (HW-ASSETS-400)

## Purpose and scope

This document is the **curated artifact index** for the Sense360 240v PSU
board, revision R4 (`S360-400-R4`). It records the schematic evidence
delivered under HW-ASSETS-400, classifies it against the
[Hardware Artifact Policy (archived)](../../archive-index.md), and
inventories what is still missing before any pin-map, package, product,
WebFlash, release, or import work can proceed.

This index is **documentation only**. It does not:

- change any value in [`config/hardware-catalog.json`](../../../config/hardware-catalog.json),
  [`config/product-catalog.json`](../../../config/product-catalog.json),
  [`config/webflash-builds.json`](../../../config/webflash-builds.json),
  or [`config/webflash-compatibility.json`](../../../config/webflash-compatibility.json),
- change any product YAML under [`products/`](../../../products/),
  any WebFlash wrapper under
  [`products/webflash/`](../../../products/webflash/), or any package
  YAML under [`packages/`](../../../packages/),
- change any workflow under `.github/workflows/`, any script under
  `scripts/`, any test under `tests/`, any component under
  `components/`, any header under `include/`, or `.gitignore`,
- mark `S360-400-R4` `verified` — `schematic_status` stays
  `cataloged_unverified`,
- add or change `schematic_file` for `S360-400` in
  [`config/hardware-catalog.json`](../../../config/hardware-catalog.json),
- change [`packages/hardware/power_240v.yaml`](../../../packages/hardware/power_240v.yaml)
  from `package-yaml-pending`,
- promote `S360-400` to `preview` / `stable` / `production`,
- add a `PWR`-bearing entry to
  [`config/webflash-builds.json`](../../../config/webflash-builds.json),
- regenerate firmware, create a release, or change any WebFlash
  manifest / import,
- **change the mains-voltage UK / EU compliance status of `S360-400`
  (COMPLIANCE-001;
  [`docs/compliance/mains-voltage-uk-eu-assessment.md`](../../compliance/mains-voltage-uk-eu-assessment.md)).**

> **S360-400 stays mains-compliance-gated.** Committing this schematic
> and creating this artifact index does **not** clear COMPLIANCE-001
> for `S360-400`, does **not** clear creepage / clearance / thermal /
> EMI/EMC / safety review for the assembled board, and does **not**
> imply that any `PWR`-bearing product may be assembled, shipped,
> previewed, or imported. **A schematic is not a compliance artefact.**
> See [Compliance / safety note](#compliance--safety-note).

The Release-One shipping configuration remains
`Ceiling-POE-VentIQ-RoomIQ` with artifact
`Sense360-Ceiling-POE-VentIQ-RoomIQ-v1.0.0-stable.bin` and tag
`v1.0.0`. The LED preview path (`Ceiling-POE-VentIQ-RoomIQ-LED`) is
unchanged. FanTRIAC (`Ceiling-POE-VentIQ-FanTRIAC-RoomIQ`) stays
`status: blocked`, `blocker: HW-005`, `webflash_build_matrix: false`.
The four `legacy-compatible` `*-pwr` Core variants under
[`products/`](../../../products/) stay `legacy-compatible` and
`webflash_build_matrix: false`.

## Policy reference

This index applies
[`docs/hardware/hardware-artifact-policy.md` (archived)](../../archive-index.md)
(HW-ASSETS-001). Relevant rules:

- Curated schematic PDFs may be committed directly. The S360-400-R4
  schematic PDF is committed by this PR at
  [`docs/hardware/schematics/S360-400-R4.pdf`](../schematics/S360-400-R4.pdf).
- Mac OS metadata, editor / IDE history, embedded `.git/`, vendor
  scratch exports, KiCad temporary / cache / backup files, and
  private supplier / customer data are excluded outright.
- KiCad source, KiCad PCB, KiCad project metadata, BOM, CPL,
  gerbers, drill files, STEP, and board images are
  **retained-but-not-committed** under the current policy.
- Per-board indexes record file sizes and SHA256 for every artifact
  actually present in the upload. Missing files are marked
  `not provided in this upload`; checksums are never fabricated.
- **Hardware artifact availability does not imply firmware support,
  WebFlash availability, build-matrix inclusion, product-catalog
  status, mains-voltage compliance, or Release-One promotion.**
- A schematic alone is not sufficient evidence to ship mains
  hardware. The COMPLIANCE-001 mains-voltage UK / EU assessment is
  a separate gate that this PR does not change.

### What the schematic appears to contain

Observed directly from the uploaded PDF. Every claim below is sourced
from a visible label or symbol on the single schematic sheet.

- **Section title: "AC-DC REGULATION TO 5V".** One section, no
  sub-sheets. The whole circuit is drawn inside a single dashed
  boundary on the A4 sheet.
- **AC input connector `J1`** — **3-pin** (visible pins 1 / 2 / 3),
  with adjacent labels on the schematic:

  | Module `J1` pin | Net (as labelled) |
  |---:|---|
  | 1 | `LIVE` |
  | 2 | `NEUTRAL` |
  | 3 | `Earth_Protective` (with the protective-earth symbol shown below the connector) |

  Silkscreen pin-1 location and connector-keying / -orientation on the
  module side are **not** derivable from the PDF and are owed to
  `HW-PINMAP-400-FOLLOWUP`.
- **Resettable fuse `F1`** — value field reads `A250-1200`. Placed in
  series on the `LIVE` leg between `J1` pin 1 and the downstream
  protection / filter / converter network. The schematic shows it as
  a resettable fuse (PPTC / polyfuse-style symbol); the precise
  hold-current / trip-current / voltage rating implied by `A250-1200`
  is **not** further annotated on the sheet and is BOM-bound.
- **Voltage-dependent resistor `RV1`** — value field reads `10D391K`.
  Placed across the AC line (between `LIVE` and `NEUTRAL` after `F1`).
  No clamping voltage / energy rating is annotated beyond the part-
  number string; the clamp / energy spec is BOM-bound.
- **X-class capacitor `C1`** — value field reads `470nF`. Placed
  across the AC line (between `LIVE` and `NEUTRAL`) after `F1` and
  in parallel with `RV1`. The schematic shows it in the X-cap
  position; the safety class (X1 / X2) is **not** annotated on the
  schematic and is BOM-bound.
- **AC/DC converter `PS1`** — value field reads `HLK-10M05`. Pinout
  as labelled on the schematic:

  | `PS1` pin | Function (as labelled) |
  |---:|---|
  | 1 | `AC(L)` |
  | 2 | `AC(N)` |
  | 3 | `-VO` |
  | 4 | `+VO` |

  The primary side connects to the post-fuse / post-MOV / post-X-cap
  `LIVE` and `NEUTRAL` nets; the secondary side connects to the
  output-filter network and to the `J2` output. No on-board boost or
  secondary regulator is visible.
- **Output filter network.** Four capacitors are shown on the +VO /
  GND rail forming a parallel filter bank between `PS1` pin 4 (`+VO`)
  and `PS1` pin 3 (`-VO` → `GND`):

  | Reference | Value |
  |---|---|
  | `C5` | `100uF` |
  | `C6` | `10u` |
  | `C7` | `100n` |
  | `C8` | `100uF` |

  A `FILTER` label is shown above this cap bank on the sheet. No
  inductor / common-mode choke / second-stage filter is visible.
- **Output connector `J2`** — **2-pin**. Pins as labelled on the
  schematic:

  | Module `J2` pin | Net (as labelled) |
  |---:|---|
  | 1 | `+5VP` |
  | 2 | `GND` |

  Silkscreen pin-1 location and connector-keying / -orientation on the
  module side are **not** derivable from the PDF and are owed to
  `HW-PINMAP-400-FOLLOWUP`.
- **Mounting holes `H1`, `H2`, `H3`, `H4`.** Four mounting-hole
  symbols are placed below the main converter block. Each appears
  with no net connection on this sheet. Whether they are
  electrically floating or are part of any mechanical bonding /
  earthing scheme is **not** derivable from the schematic alone.
- **Single-page schematic confirmed.** `Sheet: /`, `Id: 1/1`, A4,
  KiCad 10.0.3.
- **Title-block fields are blank.** `Title` / `Date` / `Rev` are
  empty in the rendered title block. The file-name field reads
  `S360-400-R4.kicad_sch`.
- **What the schematic does NOT visibly contain.**
  - No Y-class capacitors between the secondary `-VO` / `GND` and
    `Earth_Protective` (the `Earth_Protective` net appears only at
    the `J1` pin 3 terminal and does not visibly traverse the
    converter's isolation barrier on the sheet).
  - No common-mode or differential-mode line filter inductors
    between `F1` and `PS1`.
  - No on-board secondary regulator, boost converter, LDO, or
    point-of-load device.
  - No on-board indicator LED for "mains present" or "+5V present".
  - No I²C / UART / DAC / SPI / GPIO circuitry — the board is a
    bare AC/DC supply, not a smart power module.
  - No thermal cut-out, NTC inrush limiter, or relay-isolated
    soft-start visible on the sheet.
  - No Core-side mating connector pinout (the 240 V PSU is an
    off-board module; the Core schematic
    [`s360-100-r4-core.md`](../s360-100-r4-core.md) does **not**
    enumerate a mating connector for `S360-400`).
