# S360-210-R4 — Sense360 AirIQ Hardware Reference

## Purpose

This document is the schematic-backed pin, connector, and net reference for the
**Sense360 AirIQ** air-quality sensor board, revision **R4** (`S360-210-R4`).
It exists so that future firmware audits, future product onboarding for any
`AirIQ`-bearing config string, and any later WebFlash taxonomy work have a
single citable source of truth for what the silicon and copper on this board
actually do.

This document is hardware reference only. It does **not** by itself prove that
any ESPHome YAML in this repository is correct. It does **not** add `AirIQ` to
the Release-One config string, change WebFlash build behaviour, or promote any
catalog entry. The `AirIQ` token remains mutually exclusive with `VentIQ` per
[`config/webflash-compatibility.json`](../../config/webflash-compatibility.json)
(`rules.airiq_and_ventiq_mutually_exclusive: true`).

This PR (HW-007) commits the module-side schematic PDF and this standalone
reference doc. The machine-readable `schematic_status` field for `S360-210` in
[`config/hardware-catalog.json`](../../config/hardware-catalog.json) is **not**
changed by this PR — that flip is owned by HW-008.

## Main components (visible on this sheet)

- **SGP41** — Sensirion VOC / NOₓ sensor. Connects to the shared I²C bus via
  the Core's `J9` connector.
- **SCD41** — Sensirion photoacoustic CO₂ sensor. Connects to the shared
  I²C bus.
- **MICS-4514** — multi-gas (CO, NO₂) MEMS sensor. Driven by an **STM8**
  helper MCU on a dedicated sub-sheet; the STM8 bridges the MICS analog
  front-end to the shared I²C bus so the ESP32-S3 host on the Core sees a
  single I²C target rather than the MICS analog interface directly.
- **STM8** — helper MCU on the MICS sub-sheet acting as the MICS-to-I²C
  bridge. Implementation details on the STM8 sub-sheet (firmware version,
  registers exposed, power sequencing) are **verify** — the schematic
  identifies the part and the bridging role only.
- **SFA40** (`U2`) — Sensirion formaldehyde (HCHO) sensor, **PCB-mounted**
  on the shared I²C bus @ `0x5D` (verified R4 BOM + schematic). This board
  does **not** carry an off-board SFA40 connector; the sensor is populated
  directly. (Earlier revisions of this doc described SFA40 as an off-board
  connector — corrected under AIRIQ-HW-RECONCILE-001; see
  [Firmware mapping notes](#firmware-mapping-notes).)
- **SPS30** connector — particulate-matter (PM₁ / PM₂.₅ / PM₄ / PM₁₀)
  sensor connector on `J2`. The SPS30 itself lives off-board on a Sensirion
  SPS30 module (external, opt-in — not part of the S360-210 PCB).
- **Module connector to Core** — the AirIQ board plugs into the Sense360
  Core's `J9` "AirIQ Module Connector" (7-pin header on `S360-100-R4` —
  see [`s360-100-r4-core.md` §J9](s360-100-r4-core.md#j9--airiq-module-connector-7-pin)).
  Exact pin order on the AirIQ side of the connector is **verify** against
  the Core J9 table and the silkscreen on both boards.

## Module connector mating

On the Core (`S360-100-R4`), `J9` is the 7-pin "AirIQ Module Connector"
that AirIQ plugs into. The Core J9 table is the source of truth for the
Core-side pin order:

| Core J9 pin | Core J9 net          | Direction on AirIQ                                                  |
| ----------- | -------------------- | ------------------------------------------------------------------- |
| 1           | `+5V`                | Powers SPS30 and other +5 V loads (PM, HCHO, MICS heater)           |
| 2           | `+3.3V`              | Powers SGP41, SCD41, STM8 logic                                     |
| 3           | `I2C_SDA`            | Shared I²C bus — talks to SGP41, SCD41, STM8 (MICS bridge), SFA40 if I²C |
| 4           | `I2C_SCL`            | Shared I²C bus                                                      |
| 5           | `AirQ_Status_Led`    | Legacy `AirQ`-prefixed net — see Open Questions                     |
| 6           | `AirQ_Led`           | Legacy `AirQ`-prefixed net — see Open Questions                     |
| 7           | `GND`                |                                                                     |

The above is taken from the Core sheet; cross-check is **verify** against
the AirIQ schematic silkscreen. Whether `AirQ_Led` and `AirQ_Status_Led`
are actually consumed by the AirIQ board (and which on-board indicator
they drive) is **verify** — this doc does not resolve HW-002 Open
Question #4.

## On-board sensors and sub-sheets

| Sensor / part   | Interface              | Notes                                                                                                 |
| --------------- | ---------------------- | ----------------------------------------------------------------------------------------------------- |
| SGP41           | I²C (shared bus)       | Sensirion VOC index + NOₓ index. Visible on this sheet.                                              |
| SCD41           | I²C (shared bus)       | Sensirion photoacoustic CO₂. Visible on this sheet.                                                  |
| MICS-4514       | Analog → STM8 → I²C    | Driven by the STM8 helper MCU on a separate sub-sheet. The host (ESP32-S3 on Core) sees the STM8 over I²C, not the MICS analog pins directly. |
| STM8            | I²C target (to Core)   | MICS-to-I²C bridge MCU. Acts as a peripheral on the shared I²C bus.                                  |
| SFA40 (`U2`)    | I²C @ `0x5D`           | Sensirion formaldehyde, **PCB-mounted** on the shared bus. Driven by the scoped `sfa40` ESPHome component (start `0x00AC` / read `0xC0EB` per the SFA40 datasheet). |
| SPS30 connector | I²C @ `0x69` (external) | Sensirion particulate-matter module on `J2`. Off-board, opt-in only (driver in `packages/boards/s360-210-airiq-sps30.yaml`). |

The STM8 helper MCU and its I²C front-end mean the ESP32-S3 only needs to
implement an I²C client for MICS-4514 — the analog gas-sensor front-end
(heater control, ADC, MICS calibration) is handled by the STM8.

## I²C bus

All on-board I²C devices and the STM8 bridge share the single I²C bus that
enters the board through the Core `J9` connector. The Core side of that
bus has the 10 kΩ pull-ups (`R21`, `R22` on `S360-100-R4`); this AirIQ
board does not appear to duplicate them. Exact resistor placement and
values local to AirIQ are **verify**.

| I²C target on AirIQ | Address (where labelled) | Notes                                                                  |
| ------------------- | ------------------------ | ---------------------------------------------------------------------- |
| SGP41               | `0x59`                   | Sensirion default; consumed by the `sgp4x` driver.                    |
| SCD41               | `0x62`                   | Sensirion default; consumed by the `scd4x` driver.                    |
| SFA40 (`U2`)        | `0x5D`                   | Sensirion default; PCB-mounted; scoped `sfa40` driver.                |
| STM8 (MICS bridge)  | `0x60`                   | Per the owner-provided STM8 firmware spec (24-byte register block at 0x00–0x17 + a 0xFE command register); scoped `mics_stm8` driver. On-hardware bench verification pending. |

## Open questions / verification needed

These items are explicitly out of scope for HW-007 to resolve. They are
inputs to future PRs (HW-008 status flip, any future `AirIQ`-bearing
config-string PR, any future firmware audit of `airiq_*.yaml` packages).

1. **AirIQ side of the J9 connector — 1-to-7 pin order.** The Core J9
   table is committed; the AirIQ side has to match it. Confirm against the
   AirIQ silkscreen.
2. **`AirQ_Led` / `AirQ_Status_Led` reuse on AirIQ.** HW-002 Open Question
   #4 is still open. Whether these nets drive an on-board indicator on
   AirIQ, and whether VentIQ reuses the same physical lines, is not
   resolved by this PDF alone.
3. **STM8 firmware identity and I²C register map.** The schematic shows
   the STM8 acting as the MICS-to-I²C bridge but does not state the
   firmware version or the I²C register layout it exposes.
4. **MICS-4514 heater power source.** The MICS-4514 needs a heater
   supply; whether the heater is driven from the `+5 V` rail or via a
   regulated rail on the AirIQ board is **verify**.
5. **SFA40 connector wiring.** Whether the SFA40 connector carries the
   I²C bus, a separate UART, or both, is **verify** against the SFA40
   datasheet and the AirIQ silkscreen.
6. **SPS30 connector wiring.** SPS30 supports both UART and I²C; the
   connector's interface is **verify**.
7. **AirIQ-side decoupling and pull-ups.** Per-device 100 nF decoupling
   caps and any local I²C pull-ups on AirIQ are not exhaustively
   captured here; the Core-side I²C pull-ups (`R21`, `R22` on
   `S360-100-R4`) are the authoritative bus pull-ups.

## Firmware mapping notes

This document is hardware reference only. It does **not** by itself prove
that any ESPHome YAML is correct.

The relevant firmware packages today (logical, not pin-bound to a verified
schematic) are the `airiq_*.yaml` packages under
[`packages/expansions/`](../../packages/expansions/) and
[`packages/features/`](../../packages/features/), plus the canonical board
package [`packages/boards/s360-210-airiq.yaml`](../../packages/boards/s360-210-airiq.yaml).

**AIRIQ-HW-RECONCILE-001 update.** The canonical board package has been
reconciled against this schematic and the R4 BOM: it now probes the fitted
stack — SGP41 (`0x59`), SCD41 (`0x62`), the **PCB-mounted SFA40** (`0x5D`,
scoped `sfa40` driver) and the **MICS-4514/STM8 bridge** (`0x60`, scoped
`mics_stm8` driver) — and no longer instantiates a BMP390 (no pressure part
exists on this schematic/BOM) or unconditionally probes the external SPS30
(now an opt-in overlay). MICS is a diagnostic reducing/oxidising ratio
surface only; no CO/NO2 concentration is claimed. The STM8 register map,
address (`0x60`) and firmware version (`0x24`) come from the owner-provided
STM8 firmware specification; **on-hardware bench verification of the fitted
MICS/STM8 and SFA40 stages, and physical CPL/silkscreen population evidence
(HW-PINMAP-210-FOLLOWUP), remain pending** — this reconciliation is
source + datasheet + compile evidence only, never a bench/compliance claim.

`AirIQ` is **not** in Release-One. The current Release-One config string
is `Ceiling-POE-VentIQ-RoomIQ`. `AirIQ` and `VentIQ` are mutually
exclusive per
[`config/webflash-compatibility.json`](../../config/webflash-compatibility.json).
Any future `AirIQ`-bearing config string is a separate product decision
and a separate PR; it requires its own catalog entry, build-matrix entry,
release-notes draft, and onboarding-gate clearance per
[`docs/product-onboarding.md` (archived)](../archive-index.md).
