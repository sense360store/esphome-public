# S360-100-R4 — Sense360 Core Hardware Reference

## Purpose

This document is the schematic-backed pin, connector, and net reference for the
**Sense360 Core** main board, revision **R4** (`S360-100-R4`). It exists so that
firmware audits — in particular the upcoming Release-One YAML audit — have a
single, citable source of truth for what the silicon and copper actually do.

This document is hardware reference only. It does **not** by itself prove that
any ESPHome YAML in this repository is correct. Firmware mappings must be
verified separately by comparing this document against the package and product
YAML (see [Firmware mapping notes](#firmware-mapping-notes)).

## Main components

- **U6** — `ESP32-S3-WROOM-1-N16R8` MCU module (16 MB flash, 8 MB octal-SPI PSRAM).
- **U3** — `SX1509BIULTRT` 16-channel I²C GPIO / PWM expander.
- **U1** — `RT8059GJ5` step-down buck converter (5 V → 3.3 V).
- **U2A** — `74LVC1G07SE-7` open-drain buffer used as `LED_DATA` driver / level shifter.
- **J8** — USB-C 2.0 receptacle (data + VBUS).
- **J2** — `PoE_ACDC` 2-pin power inlet (fed by external PoE PSU module S360-410).
- **Q2** — `Si2319CDS` p-channel MOSFET, ideal-diode-OR between USB VBUS and PoE_ACDC into +5 V.
- **Q4** — `SI2302S` n-channel MOSFET on the LED level-shifter section.
- **R8** — 330 Ω series resistor on the WS2812B data line.
- **R12 / R13 / R21 / R22** — 10 kΩ pull-ups (reset, expander INT, I²C SDA/SCL).
- **R5 / R6** — 70.6 kΩ / 15.1 kΩ buck feedback divider (per schematic note,
  computed Vout ≈ 3.4 V due to manufacturer compensation, in practice ≈ 3.3 V).
- **L1** — 2.2 µH buck inductor.
- **D5** — yellow LED, +5 V rail indicator.
- **D6** — blue LED, +3.3 V rail indicator.
- **D12** — `1N5819WS` Schottky diode (USB VBUS path).
- **SW3** — Boot/IO0 tactile switch (pulled to +3.3V via R40 10 k).
- **SW4** — Reset tactile switch (drives `EN` via R12 10 k pull-up + C44 1 µ).
- **TP10 / TP13 / TP14** — JTAG test points (`MTMS`, `MTDO`, `MTDI`).
- Module connectors: `J3` (WS2812B LED), `J4` (Relay), `J6` (12 V PWM fan),
  `J7` (GP8403 fan), `J9` (AirIQ module), `J10` (Presence/Comfort module —
  RoomIQ), `J13` (FAN), `J15` (TRIAC fan module).

## Connector / pin strategy — Core is the hub; tach / pulse counter is native

`S360-100` is the **central hub** for the room / module stack: every
module connector listed in [§Connectors](#connectors) and
[§Module connector summary](#module-connector-summary) derives its
signals from the Core board. Two architectural constraints follow from
that role and are recorded in detail in
[`docs/hardware/s360-100-native-tach-pulse-strategy.md`](s360-100-native-tach-pulse-strategy.md)
(S360-100-NATIVE-TACH-PULSE-001):

- **Tach / pulse-counter inputs must terminate on native ESP32-S3
  GPIO.** Any `sensor: platform: pulse_counter`, any RPM input, and
  any pulse-counting line must land on an interrupt-capable
  `InternalGPIOPin`.
- **The SX1509 I/O expander (U3) must NOT be used for tach /
  pulse-counter inputs.** PWM-drive **output** through the SX1509
  (`output: platform: sx1509`) is a separate capability and remains
  supported; tach **input** support is independent and is **not**
  granted by PWM-drive support.

The current canonical S360-100-R4 schematic
([`schematics/S360-100-R4.pdf`](schematics/S360-100-R4.pdf), SHA256
`4c9e8b06d129fbb55f61e143b648e03762d06cb4dc67fe3120c268cd3a4bdf16`)
now terminates the per-fan `Pul_Cou1..4` and `TachPMW1..4` lines
directly at native ESP32-S3 module pins — the schematic-side hardware
re-route called out in the prior R4 snapshot is delivered by the
refreshed sheet. The final native-GPIO allocation is recorded in
[§ S360-100-TACH-GPIO-ALLOCATION-001 — native ESP32 GPIO allocation
for FanPWM tach inputs](#s360-100-tach-gpio-allocation-001--native-esp32-gpio-allocation-for-fanpwm-tach-inputs)
below; the pin-allocation table in
[`docs/hardware/s360-100-core-architecture.md` § Pin allocation table](s360-100-core-architecture.md#pin-allocation-table--native-esp32-s3-gpio-termination)
is the canonical Core-architecture-index view of the same allocation.
The legacy SX1509-routed text in the
[§Fan / driver outputs](#fan--driver-outputs) table below is the
**pre-refresh** snapshot kept as historical context per the
[R4 refresh note](#schematic-source) above; treat the
S360-100-TACH-GPIO-ALLOCATION-001 section as the source of truth for

## S360-100-TACH-GPIO-ALLOCATION-001 — native ESP32 GPIO allocation for FanPWM tach inputs

**Status: documentation-only — schematic-proven, firmware-binding and
bench-measured RPM not claimed.** This section pins the **final** native
ESP32-S3 GPIO allocation for the FanPWM tach / pulse-counter inputs
`Pul_Cou1..4` (plus the shared `TachIO` passthrough and the
`TachPMW1..4` PWM-drive outputs) from the canonical
[`schematics/S360-100-R4.pdf`](schematics/S360-100-R4.pdf) sheet. It is
the **next blocker** after `S360-100-NATIVE-TACH-PULSE-001` and it
records, on the Core-side hardware reference, the per-fan GPIO numbers
the architectural rule requires.

The allocation is **schematic-proven** by the committed R4 sheet (SHA256
`4c9e8b06d129fbb55f61e143b648e03762d06cb4dc67fe3120c268cd3a4bdf16`,
837,443 bytes, KiCad E.D.A. 10.0.3). It is **not** bench-verified, **not**
firmware-bound, and **not** measured-RPM-validated by this document.

## I2C bus

| Net       | ESP32 pin | Pull-up    | Devices / connectors that share the bus                                              |
| --------- | --------- | ---------- | ------------------------------------------------------------------------------------ |
| `I2C_SDA` | IO48      | R22 10 k   | SX1509 (U3, on-board), J7 GP8403 fan, J9 AirIQ, J10 RoomIQ                           |
| `I2C_SCL` | IO45      | R21 10 k   | SX1509 (U3, on-board), J7 GP8403 fan, J9 AirIQ, J10 RoomIQ                           |

The SX1509 GPIO/PWM expander sits on the same shared I²C bus as the external
modules. Its `INT` line is `expander_int` (ESP32 IO17) with R13 10 k pull-up,
and its `RESET` is driven by `RST1`.

## UART / radar-related signals

The Core board exposes three logical UARTs at the connector level:

| Logical UART    | ESP32 pins                  | Destination                                          | Purpose                                  |
| --------------- | --------------------------- | ---------------------------------------------------- | ---------------------------------------- |
| UART0 (boot log)| TXD0 (pin 37) / RXD0 (pin 36) | J7 GP8403 fan (also used as boot log on USB)       | GP8403 DAC fan driver UART               |
| `Hi-Link`       | IO2 (TX) / IO1 (RX)         | J10 RoomIQ Pin 7 / Pin 6                             | Hi-Link mmWave radar UART (RoomIQ)       |
| `SEN0609`       | IO5 (TX) / IO4 (RX)         | J10 RoomIQ Pin 4 / Pin 3                             | DFRobot SEN0609 radar UART (RoomIQ)      |

`out(gpio6)` (ESP32 IO6 → J10 pin 5) is an additional auxiliary line carried
to the RoomIQ connector but is not, on this sheet, dedicated to a UART.
