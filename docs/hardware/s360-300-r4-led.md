# S360-300-R4 — Sense360 LED Hardware Reference

## Purpose

This document is the schematic-backed reference for the **Sense360 LED**
board, revision **R4** (`S360-300-R4`). It exists so that future audits and
any future LED-bearing config string have a single citable source of truth
for what the LED board itself carries.

This document is hardware reference only. It does **not** by itself prove
that any ESPHome YAML in this repository is correct. The HW-007 PR commits
the module-side schematic PDF and this standalone reference doc.

The machine-readable `schematic_status` field for `S360-300` in
[`config/hardware-catalog.json`](../../config/hardware-catalog.json) is
**not** changed by this PR — the status promotion from `cataloged_unverified`
is deferred to HW-008.

> **Sense360 LED is not in Release-One.** The Release-One config string is
> `Ceiling-POE-VentIQ-RoomIQ` and carries **no** `LED` token. The
> Release-One YAML
> [`products/sense360-ceiling-poe-ventiq-roomiq.yaml`](../../products/sense360-ceiling-poe-ventiq-roomiq.yaml)
> omits LED package includes on purpose. HW-007 commits LED schematic
> evidence; it does **not** add `LED` to the Release-One config string,
> the Release-One artifact name, the WebFlash build matrix, or the product
> catalog. A future LED-bearing config (e.g. `Ceiling-POE-VentIQ-RoomIQ-LED`)
> is a separate product PR.

## Main components (visible on this sheet)

- **Connector `J1`** — 3-pin module connector. Pins (per the schematic):
  - `LED_DATA` — WS2812B data input
  - `+5V` — LED power
  - `GND` — ground
- **WS2812B LED chain** — addressable RGB LEDs driven from the connector's
  `LED_DATA` line. Exact LED count and chain order are **verify** against
  the silkscreen.

The Core side of `LED_DATA` is the `74LVC1G07` (`U2A`) open-drain buffer
plus the 330 Ω series resistor (`R8`) on `S360-100-R4`, terminating at the
Core's `J3` 3-pin LED connector — see
[`s360-100-r4-core.md` LED output](s360-100-r4-core.md#led-output).
The Sense360 LED board's `J1` mates with the Core's `J3` either directly
or via a short harness. The harness identity (direct mate vs cable) is
**verify**.

## Connector `J1` — module connector (3-pin)

| Pin | Net        | Notes                                                           |
| --- | ---------- | --------------------------------------------------------------- |
| 1   | `LED_DATA` | WS2812B data input. Mates with the Core `J3` `LED_DATA_3V3` pin (after the `U2A` buffer + `R8` 330 Ω on Core). |
| 2   | `+5V`      | LED power. Mates with the Core `J3` `+3.3V` or `+5V` rail — **verify** against the Core J3 capture (`s360-100-r4-core.md` records Core J3 pin 1 as `+3.3V`; a `+5V` mismatch on the silkscreen would be a separate Open Question for HW-008 / silkscreen verification). |
| 3   | `GND`      | Common ground.                                                  |

The exact 1-to-3 pin order on `J1` is **verify** against the silkscreen.
The Core `J3` table in
[`s360-100-r4-core.md` §J3](s360-100-r4-core.md#j3--ws2812b-led-connector-3-pin)
records the mating side; both tables should agree on a real board, and
any disagreement is a verification gap, not an HW-007 finding.

## LED chain

The schematic shows a chain of WS2812B addressable LEDs driven from the
`LED_DATA` pin of `J1`. Each WS2812B integrates an RGB LED + serial-data
controller, so the chain is cascaded `DOUT → DIN` through successive LEDs.
Exact LED count on the production S360-300-R4 board is **verify** against
the silkscreen and the PCB layout — this doc does not state a count.

## What this schematic proves

- The Sense360 LED board has its own module-side schematic, now committed
  to this repo at
  [`schematics/S360-300-R4.pdf`](schematics/S360-300-R4.pdf).
- The board exposes a 3-pin module connector (`J1`) carrying `LED_DATA`,
  `+5V`, and `GND`.
- The on-board LED implementation is a WS2812B chain driven from
  `LED_DATA`.

## What this schematic does **not** prove

- It does **not** prove that Sense360 LED is part of Release-One. The
  Release-One config string `Ceiling-POE-VentIQ-RoomIQ` carries no `LED`
  token. The Release-One YAML omits LED package includes on purpose.
- It does **not** resolve the `LED_DATA` source-pin discrepancy at the
  package level:
  - [`packages/hardware/led_ring_ceiling.yaml`](../../packages/hardware/led_ring_ceiling.yaml)
    drives WS2812B on `GPIO14`.
  - The Core schematic (`S360-100-R4`) shows `LED_DATA` originating at
    ESP32-S3 `IO38`, with `IO14 = SCS` (peripheral SPI chip-select).
  - HW-007 commits the LED-side schematic, not a new Core-side trace.
    Resolving `GPIO14` (package) vs `IO38` (Core schematic) requires
    either correcting the package or proving an alternative Core-side
    routing — both are out of scope here.
- It does **not** add `LED` to the Release-One config string.
- It does **not** change the WebFlash build matrix or the product
  catalog.
- It does **not** flip the JSON `schematic_status` field for `S360-300`.

## Open questions / verification needed

These items are explicitly out of scope for HW-007.

1. **`J1` pin 2 rail identity (`+5V` vs `+3.3V`).** The Core `J3` table
   records the rail as `+3.3V`; the Sense360 LED board may operate
   WS2812B from `+5V` instead. Confirm against the silkscreen on both
   boards.
2. **LED count on the chain.** Not stated in this doc; **verify** against
   the silkscreen.
3. **`LED_DATA` source-pin reconciliation at the package level.**
   `GPIO14` (package) vs `IO38` (Core schematic). This is a package-vs-
   Core discrepancy, not an S360-300 issue, and is the responsibility of
   any future LED-bearing config-string PR.
4. **Harness between Core `J3` and LED `J1`.** Direct mate vs short
   cable / pigtail is **verify**.
5. **LED protection components on the LED board.** Series resistors,
   capacitor across `+5V`/`GND`, or ESD protection local to the LED
   board are not exhaustively captured here. The Core side carries the
   330 Ω series on `LED_DATA` (`R8`) and the 74LVC1G07 (`U2A`) buffer;
   any LED-side additions are **verify**.

## Firmware mapping notes

This document is hardware reference only.

The Sense360 LED package today is
[`packages/hardware/led_ring_ceiling.yaml`](../../packages/hardware/led_ring_ceiling.yaml),
which drives WS2812B on `GPIO14`. The Core schematic puts `LED_DATA` on
`IO38`. HW-007 does not reconcile that discrepancy; reconciliation belongs
to a future LED-bearing config-string PR.

The Release-One YAML
[`products/sense360-ceiling-poe-ventiq-roomiq.yaml`](../../products/sense360-ceiling-poe-ventiq-roomiq.yaml)
does **not** include the LED package — the WebFlash config string
`Ceiling-POE-VentIQ-RoomIQ` carries no `LED` token, so the binary built
from that YAML does not include LED firmware.
