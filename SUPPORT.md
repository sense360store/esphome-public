# Getting help with Sense360 firmware

Use the channel that matches your situation — it gets you to the right
people fastest.

## Something is broken (defects)

File a **GitHub Issue** on the repository that owns the problem:

- **Firmware behaviour, product YAML, sensors, automations, builds** —
  [Issues on `esphome-public`](https://github.com/sense360store/esphome-public/issues)
  (this repository). Please use the issue templates: flashing problem,
  hardware fault, configuration question, or feature request.
- **The WebFlash site itself** (browser flashing fails, wrong firmware
  listed, site errors) —
  [Issues on `WebFlash`](https://github.com/sense360store/WebFlash/issues).

## Questions and community help

Ask in [GitHub Discussions on `esphome-public`](https://github.com/sense360store/esphome-public/discussions)
— configuration questions, Home Assistant integration, "is this expected?",
sharing setups. Discussions is the community Q&A home for both repositories.

## Orders, shipping, returns, and warranty

Order and warranty matters are **not** handled on GitHub. Use the contact
page at [mysense360.com](https://mysense360.com) — include your order number.

## Before you file

- Check [`docs/release-channels.md`](docs/release-channels.md) to confirm
  which channel (stable / preview / experimental) your device is running —
  support expectations differ per channel.
- Preview and experimental firmware is provided for testers and self-builders
  **at your own risk**; the stable channel is the supported customer
  baseline.
- Most customers should flash via
  [WebFlash](https://sense360store.github.io/WebFlash/); this repository is
  the manual / custom firmware path for advanced users.
